Practice 1

1. Context

In many algorithm problems, we often need to answer this question:

Does a value already exist in our data?

Searching for a value in a normal array is slow.

Time complexity:

O(n)

So when algorithms require many searches, this becomes inefficient.

This is why we use a Hash Map.

A Hash Map allows us to check if something exists in:

O(1)

which is much faster.

2. The Problem Example: Two Sum

We are given:

nums = [2,7,11,15]
target = 9

We must find two numbers whose sum equals the target.

Expected result:

2 + 7 = 9

Return their indices.

[0,1]

Important rule:

We cannot use the same index twice.

3. Brute Force Idea (Slow)

First idea: try every pair of numbers.

Example:

2 + 7  = 9 ✓
2 + 11 = 13
2 + 15 = 17
7 + 11 = 18
7 + 15 = 22
11 + 15 = 26

Algorithm idea:

for each number i
    for each number j
        check nums[i] + nums[j]

Time complexity:

O(n²)

This becomes very slow for large arrays.

4. Better Idea

Instead of checking every pair, we ask a smarter question.

For each number num, we ask:

What number do I need to reach the target?

Formula:

needed = target - num

Example:

target = 9
num = 2
needed = 7

So instead of searching for a pair directly, we ask:

Have we already seen 7?

If yes → we found the solution.

5. Using a Hash Map

A hash map stores values we already visited.

Structure:

value → index

Example while iterating through the array.

Step 1

num = 2
needed = 9 - 2 = 7

Hash map:

{}

7 not found.

Store the current number:

{2:0}

Step 2

num = 7
needed = 9 - 7 = 2

Check hash map:

{2:0}

2 exists.

So:

nums[0] + nums[1] = 9

Return:

[0,1]

6. Why This Is Faster

Hash map lookup:

O(1)

We iterate once through the array:

O(n)

Total complexity:

O(n)

Instead of:

O(n²)

This is a huge improvement.

Solution : 

public class Solution {
    public int[] TwoSum(int[] nums, int target) {
        var map = new Dictionary<int, int>(nums.Length);

        for (int i = 0; i < nums.Length; i++)
        {
            int needed = target - nums[i];

            if (map.TryGetValue(needed, out int index))
                return new[] { index, i };

            map[nums[i]] = i;
        }

        return Array.Empty<int>();
    }
}

Practice 2

1. Context

In many problems, we need to detect duplicates.

The goal is usually:

Detect the duplicate as fast as possible.

2. The Problem

We are given a string:

s = "abccba"

We must return:

The first character that appears twice

Important:

"First" means the first one whose second occurrence appears earliest.

Example:

a b c c b a
      ↑

The first duplicate is:

c

because c appears again before b or a.

3. Brute Force Solution (Slow)

The first solution checks each character with all previous characters.

Example idea:

for each character c
    check all characters before it

Example with:

s = "abccba"

Step by step:

a → check before → none
b → check before → a
c → check before → a,b
c → check before → a,b,c → match found

So we return:

c

But the algorithm uses two loops.

for i
   for j

Time complexity:

O(n²)

This becomes slow for large strings.

4. Better Idea

Instead of checking the string again and again, we remember what we already saw.

We use a Set.

A set stores unique elements and allows very fast lookup.

Operations:

add(element)
contains(element)

Both take:

O(1)

5. Efficient Solution (Using a Set)

Algorithm idea:

  1. Create an empty set
  1. Go through the string
  1. For each character:
    • check if it already exists in the set
    • if yes → return it
    • if no → add it to the set

Example:

s = "abccba"

Step 1

seen = {}

Step 2

Character:

a

Check set:

{}

Not found → add it

{a}

Next character

b

Set:

{a}

Not found → add it

{a,b}

Next character

c

Set:

{a,b}

Not found → add it

{a,b,c}

Next character

c

Set:

{a,b,c}

Found → duplicate detected.

Return:

c

6. Time Complexity

We iterate through the string once.

Each operation in the set is:

O(1)

So total time:

O(n)

Much faster than:

O(n²)

7. Space Complexity

We store characters inside the set.

Worst case:

{a,b,c,d,e,f,...}

So memory grows with input size.

O(n)

If we only allow English letters, there are only:

26 characters

So the set can never grow beyond 26.

In that case, space becomes:

O(1)

because it is constant.

Solution 

public class Solution {
    public char RepeatedCharacter(string s) {
        HashSet<char> seen = new HashSet<char>();

        foreach(char c in s){
            if(seen.Contains(c)){
                return c;
            }
            seen.Add(c);
        }

        return '\0';
    }
}

Practice 3

1. The Problem

We are given an array:

nums = [1,3,5,2,4]

We must find numbers x such that:

x + 1 NOT in nums
x - 1 NOT in nums

Meaning:

The number has no neighbors.

2. Example

Example array:

nums = [1,3,5,2,4,8]

Let’s check each number.

Check 1

1 + 1 = 2

2 exists → not valid.

Check 3

3 - 1 = 2
3 + 1 = 4

Both exist → not valid.

Check 5

5 - 1 = 4

4 exists → not valid.

Check 8

8 - 1 = 7
8 + 1 = 9

Neither exists.

So:

8 is valid

Answer:

[8]

3. Key Idea (Why Use a Set)

If we use an array, checking if something exists requires scanning.

Example:

Is 7 in nums ?

We must search the whole array.

Cost:

O(n)

But if we convert the array into a Set:

set = {1,2,3,4,5,8}

Then checking becomes:

set.contains(7)

Cost:

O(1)

Much faster.

4. Algorithm Idea

Step 1

Convert the array into a set.

numsSet = {all numbers}

Step 2

Iterate through the numbers.

For each number x, check:

if (x + 1 NOT in set) AND (x - 1 NOT in set)

Step 3

If true → add it to the result.

5. Why Convert to a Set First?

This is called pre-processing.

We prepare the data first so that later operations become faster.

6. Time Complexity

Step 1

Building the set:

O(n)

Step 2

Loop through numbers:

O(n)

Each lookup:

O(1)

Total:

O(n)

7. Space Complexity

The set stores all numbers.

If there are n numbers:

O(n)

memory.

8. Important Interview Pattern (Very Important)

When your algorithm repeatedly checks if something exists in a list, consider using a Set or Hash Map.

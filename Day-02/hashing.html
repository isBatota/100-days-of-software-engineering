1. What is a Data Structure

A data structure is simply a way to organize data so we can use it efficiently.

Example:

If you store numbers:

[3, 8, 10, 25]

This is a list / array.

2. Interface vs Implementation

Interface = HOW you use it

It is the operations available.

Example with a list:

You can do:

append(5)
remove(3)
update(index, value)

Example in JavaScript:

arr.push(10)

You use the list like this.

You don't care how it works internally.

Implementation = HOW it works internally

This is the actual code behind the scenes.

Example:

When you do:

arr.push(10)

Internally the language might:

  1. Check if memory is full
  1. Allocate more memory
  1. Move elements
  1. Insert the new value

But you don't see that.

You only use the interface.

Example analogy:

Interface = steering wheel of a car

Implementation = engine mechanics

You drive the car without knowing how the engine works.

3. Hashing (Main Idea of the Chapter)

Now we arrive to the important concept.

Hashing is used in HashMap / Dictionary / Set.

Example:

{ "apple": 5 }
{ "banana": 3 }
{ "orange": 7 }

But computers don't use strings directly.

They convert them into numbers.

This is called:

hashing

4. What is a Hash Function

A hash function converts data into a number.

Example:

hash("apple") → 42
hash("banana") → 17
hash("orange") → 9

Important rule:

Same input → same output

hash("apple") always = 42

5. Example Hash Algorithm

Suppose we want to hash the word:

"abc"

Step 1

Create a variable:

total = 0

Step 2

Convert each letter to its alphabet position:

a = 1
b = 2
c = 3

Step 3

Multiply by position in the string.

Formula:

value = alphabet_position * string_position

Positions start from 1.

For "abc" :

Character 1

a = 1
position = 1

1 * 1 = 1
total = 1

Character 2

b = 2
position = 2

2 * 2 = 4
total = 1 + 4 = 5

Character 3

c = 3
position = 3

3 * 3 = 9
total = 5 + 9 = 14

Final result:

hash("abc") = 14

6. Why Modulo (%) is Used

We cannot allow numbers to grow infinitely.

If we want our table size to be:

size = 10

We use modulo:

index = total % size

Example:

14 % 10 = 4

So we store the value in index 4.

1. Problem with Arrays

Arrays are very fast.

Example array:

index:   0   1   2   3
value:  10  20  30  40

If I want the value at index 2:

arr[2] → 30

This is O(1) (instant).

So arrays are great.

But there is a big limitation:

Array indices must be integers.

You cannot do this:

arr["apple"]

because arrays only accept numbers.

2. But Sometimes We Want This

Imagine storing fruit prices.

apple  → 2$
banana → 1$
orange → 3$

This is called key → value.

key     value
apple → 2
banana → 1
orange → 3

But arrays require numeric indices.

So how do we solve this?

3. Hash Function Solves the Problem

A hash function converts a key into a number.

Example:

hash("apple")  → 4
hash("banana") → 7
hash("orange") → 2

Now we can store them in an array:

index   value
0
1
2      orange → 3
3
4      apple → 2
5
6
7      banana → 1

So internally it is still an array.

But we can use strings as keys.

This combination is called:

Hash Map / Hash Table / Dictionary

4. Why Hash Maps Are Powerful

With hash maps we can do operations very fast.

Most operations are:

O(1)

Examples:

Add element

map["apple"] = 2

Check if key exists

if "apple" in map

Delete

delete map["apple"]

Update

map["apple"] = 5

All are constant time.

That is why hash maps are one of the most important data structures in interviews.

5. Ordered vs Unordered

Hash maps are unordered.

Example:

{
  apple: 2
  banana: 1
  orange: 3
}

When you iterate, order is not guaranteed.

You might get:

banana
apple
orange

or

orange
banana
apple

The order is not important.

6. Why Not Always Use Hash Maps?

Hash maps also have disadvantages.

1️⃣ Extra computation

Before inserting a key:

hash(key)

must be calculated.

So technically:

O(1)

but with extra cost.

2️⃣ More memory usage

Hash tables are usually bigger than the number of elements.

Example:

table size = 10000
stored elements = 10

So a lot of memory is unused.

3️⃣ Resizing is expensive

When the table becomes full:

The system must:

1. create bigger table
2. recompute hash for every key
3. reinsert everything

This is called:

rehashing

7. Important Interview Note

When we say hash map operations are:

O(1)

This means relative to number of elements.

But hashing a string takes time.

Example:

hash("apple")

The algorithm must read every character.

So complexity is:

O(m)

Where:

m = length of string

1. What is a Set

A Set is a data structure that stores unique elements only.

It is very similar to a HashMap, but it stores only values, not key-value pairs.

HashMap

Stores:

key → value

Example:

"apple"  → 5
"banana" → 3
"orange" → 7

Set

Stores only values:

apple
banana
orange

No values attached.

Example

Python example:

fruits = {"apple", "banana", "orange"}

Add element:

fruits.add("mango")

Check if element exists:

"apple" in fruits

Remove element:

fruits.remove("banana")

All these operations are usually:

O(1)

because sets use hashing internally.

2. Why Arrays Usually Cannot Be Keys

Remember:

hash(key) → number

Example:

hash("apple") → 42

If the key changes, the hash changes.

Example:

[1,2,3]

Hash computed.

But if the array becomes:

[1,2,3,4]

Now the hash should be different.

This would break the hash table.

That is why arrays usually cannot be keys.

using System;
using System.Collections.Generic;

// Declaration: C# uses Dictionary<TKey, TValue>
Dictionary<int, int> hashMap = new Dictionary<int, int>();

// Initialize with key-value pairs
Dictionary<int, int> hashMap = new Dictionary<int, int>()
{
    {1, 2},
    {5, 3},
    {7, 2}
};

// Checking if a key exists: use .ContainsKey()
hashMap.ContainsKey(1); // true
hashMap.ContainsKey(9); // false

// Accessing a value given a key
hashMap[5]; // 3

// Adding or updating a key
// If the key already exists, the value will be updated
hashMap[5] = 6;

// If the key doesn't exist yet, it will be inserted
hashMap[9] = 15;

// Deleting a key
hashMap.Remove(9);

// Get size
hashMap.Count; // 3

// Iterate over keys
foreach (int key in hashMap.Keys)
{
    Console.WriteLine(key);
}

// Iterate over values
foreach (int val in hashMap.Values)
{
    Console.WriteLine(val);
}
